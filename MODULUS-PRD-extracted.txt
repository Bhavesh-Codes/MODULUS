1. Product Overview 
 
1.1 Problem Statement 
College students lack a single cohesive platform for academic collaboration. Resources scatter 
across WhatsApp, Google Drive, and Notion. There is no shared accountability layer, no 
structured community organisation, and real-time study collaboration requires juggling Discord, 
Zoom, and separate whiteboard tools simultaneously. 
 
1.2 What MODULUS Is 
MODULUS unifies four core workflows into one product: 
• Personal Vault — private storage for all notes, files, and links. The single source of truth 
for every file on the platform. 
• Community Collaboration — shared resource libraries (referencing Vault files), task 
accountability, and discussion threads per course or group. 
• Real-Time Study Circles — ephemeral rooms with voice, video, screen share, and a 
shared Excalidraw whiteboard. 
• Focus Dashboard — Pomodoro timer, session logging, community leaderboard, and 
personal analytics charts.

8. Non-Functional Requirements 
 
Category Requirement 
Performance API responses (non-file) < 500 ms under normal load. File uploads 
streamed — must not block other requests. 
Security Passwords hashed via Supabase Auth (bcrypt). JWTs signed by Supabase. 
R2 files served via short-TTL signed URLs only — bucket never public. Zod 
validation on all API route inputs. 
RLS Supabase Row Level Security policies enforce data access at DB level. API 
is a second layer, not the only layer. 
Storage Quota enforced server-side. Client check is UX only and cannot be 
bypassed. R2 objects deleted when reference count = 0. 
Responsiveness Fully usable on mobile viewports ≥ 375 px. No horizontal scroll. 
Reliability Study Circle cleanup idempotent — server restart deletes orphan Circle 
records. R2 provides file durability. 
WebRTC TURN server (Open Relay) configured for all RTCPeerConnection 
instances. Direct connection attempted first; TURN used as automatic 
fallback. 
Privacy Vault content never exposed to other users unless explicitly shared. Task 
drill-down respects per-user opt-out setting.

10. Recommended Build Order 
 
Principle Each phase is independently testable. Never spend more than one phase on 
infrastructure nobody can interact with yet. 
 
Phase 1 — Foundation 
• Supabase project setup: tables, RLS policies for all entities. 
• Next.js app scaffold: routing, Supabase client, auth context (@supabase/ssr), protected 
routes. 
• Supabase Auth: email/password registration + verification + password reset. 
• Google OAuth: configure in Supabase dashboard, implement in app. 
• Cloudflare R2: bucket setup, upload API route, signed URL generation, delete helper. 
• Profile Setup Wizard.

Phase 2 — Personal Vault 
• File upload with quota enforcement (client + server). 
• Rich text note creation and editing. 
• Folders, sub-folders, #tags. 
• Vault UI: grid/list view, search, preview modal, delete, unshare controls. 
 
Phase 3 — Communities & Community Vault 
• Community creation (Public/Private), search, discovery. 
• Role system: Owner, Curator, Peer. Join request flow. 
• Community Vault: share from Vault, direct upload (auto-saves to Vault), community 
search, view/download, save-to-own-vault (full copy). 
• Community folders (independent of personal Vault structure). 
• Member management: promote, demote, kick, ban. 
 
Phase 4 — Collaboration Features 
• Common Tasks: create, suggest, approve, complete, progress bar, drill-down with opt-
out. 
• Discussion Threads: post types, threaded replies, upvote/downvote, lock, mark solution, 
moderation. 
 
Phase 5 — Focus Dashboard 
• FocusSession logging: label picker, Pomodoro countdown, stopwatch. 
• Community leaderboard query and display. 
• Personal analytics: Recharts bar/line chart from FocusSession records. 
 
Phase 6 — Study Circles (Most Complex) 
• Socket.io server: room lifecycle, join/leave, cleanup on server restart. 
• WebRTC signalling relay: offer/answer/ICE via Socket.io events. 
• Voice + video in UI: RTCPeerConnection per peer pair, mute/camera controls. 
• Screen sharing: getDisplayMedia, one active share per room. 
• Text chat: Socket.io broadcast, ephemeral (memory only). 
• Excalidraw whiteboard: embed component, emit onChange deltas, apply remote deltas. 
• Snapshot: exportToBlob() → R2 upload → VaultItem creation. 
• Open Relay TURN: configure ICE servers in RTCPeerConnection.
